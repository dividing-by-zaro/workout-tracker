Issue #44 — Live Activity multi-set display

Today the inline set summaries truncate to "50 lbs..." when there are 4+ sets in an exercise. Three layout options below, each preserving the brick icon for completed sets, primary-color emphasis on the current set, monospaced digits, and the "110 lbs × 8" format. Open in browser; the card width matches the lock-screen Live Activity.

Option 1 — Wrap to two rows

Smallest change. Set summaries wrap onto a second line when they overflow. Same fonts, same emphasis. Adds at most ~24pt of vertical height.

Hip Thrust (Barbell)
50 lbs × 8 110 lbs × 8 110 lbs × 8 110 lbs × 8
1:52
  • Lowest risk: same components, just a wrapping container.
  • Reads identically to today for ≤3-set exercises.
  • Card grows taller as set count grows; could hit lock-screen height limits at 8+ sets.
  • Wrapping isn't visually predictable — split point depends on label widths.

Option 2 — Current-set hero + progress pips

The set you're about to do is the only thing that needs to be readable at-a-glance. Show it big, replace the row of labels with a thin progress strip (one pip per set, current pip taller).

Hip Thrust (Barbell)
Set 4 / 4 110 lbs × 8
1:52
  • Scales gracefully to any number of sets — pips just get thinner.
  • Strongest readability of the next-up target weight × reps.
  • Constant card height regardless of set count.
  • Loses the historical "what did I do for set 1 vs set 3" glance.
  • Bigger visual departure from current design — needs a stronger product call.

Option 3 — Reps-only chips for non-current sets

Most lifters keep weight constant across sets. Drop "lbs" from completed/upcoming chips — show just ×8 — and reserve the full 110 lbs × 8 label for the current set only.

Hip Thrust (Barbell)
×8 ×8 ×8 110 lbs × 8
1:52
  • Fits comfortably on one row up to ~6 sets.
  • Keeps the inline historical row that the original design intended.
  • Minimal departure from current visual language.
  • If a lifter ramps weight (50 → 110 → 110 → 110), the chip row hides that — they'd need to remember weight changed.
  • Could conditionally show weight only when it differs from the current set; adds branching logic.

Option 4 — Stripped: red current set, no icons

No brick icons, no flame. Completed and upcoming sets are quiet gray text; the current set is red and slightly larger. Lets the typography do all the work.

  • Cleanest, calmest visual — no decorative elements competing with the timer.
  • Without icons, more horizontal room for the labels themselves.
  • Color + weight contrast alone is enough to find the current set.
  • Loses the brick-icon completion cue that's used elsewhere in the app — small consistency cost.
  • Still wraps/truncates at high set counts; pair with Option 1's wrap or Option 3's chip if needed.
